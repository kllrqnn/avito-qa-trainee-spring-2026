# Отчет о найденных дефектах (Bug Report)

## 1. Дефекты API (Backend)
---
## BUG-1. Нарушение формата JSON-ответа 

* **Краткое описание:** При создании объявления сервер возвращает ID внутри текстовой строки в поле status, а не в отдельном поле id. 
* **Фактический результат:** `{"status": "Сохранили объявление - <uuid>"}`. 
* **Ожидаемый результат:** `{"id": "<uuid>", "status": "success"}`. Идентификатор должен передаваться в отдельном поле для корректного парсинга автоматизированными системами. 
* **Шаги воспроизведения:** 1. Подготовить валидный JSON-объект для создания объявления. 
                            2. Выполнить POST запрос на `/api/1/item`. 
* **Серьезность:** Критический. 
* **Окружение:** https://qa-internship.avito.com, macOS, Python 3.12. 

---

## BUG-2. Отсутствие валидации отрицательной цены

* **Описание:** Сервер возвращает HTTP 200 на запрос с отрицательной ценой. 
* **Фактический результат:** Статус-код 200 OK, объявление создано. 
* **Ожидаемый результат:** HTTP 400 Bad Request. 
* **Шаги воспроизведения:** 1. Отправить POST запрос на эндпоинт `/api/1/item`. 
                            2. В теле запроса передать корректный JSON, но с отрицательной ценой:  

```json
{
"name": "Negative Price Item",
"price": -100,
"sellerId": 999999,
"statistics": {
    "contacts": 1,
    "likes": 1,
    "viewCount": 1
    }
}
```
* **Серьезность:** Минорный. 
* **Окружение:** https://qa-internship.avito.com, macOS, Python 3.12.                 

---

## BUG-3. Расхождение в именовании полей

* **Описание:** В объекте статистики поле лайков называется `likes`, тогда как в первичной документации/коллекции оно указано как `like`. 
* **Фактический результат:** Ошибка 400 при использовании `like`. 
* **Ожидаемый результат:** Унификация нейминга. 
* **Шаги воспроизведения:** 1. Отправить POST запрос на эндпоинт `/api/1/item`. 
                            2. В теле запроса передать корректный JSON, используя поле like внутри объекта статистики:  

```json
{
"name": "Test Item",
"price": 100,
"sellerId": 123456,
"statistics": {
    "contacts": 10,
    "like": 5,
    "viewCount": 100
    }
}
```
* **Серьезность:** Мажорный. 
* **Окружение:** https://qa-internship.avito.com, macOS, Python 3.12.

## 2. Дефекты интерфейса и логики поиска (UI/UX)
---
## P0 — Критические баги

- **P0 — Некорректная выдача по фильтру породы.** В поиске выбрана порода «Персидская», в результатах на третьей позиции отображается бенгальские коты, а ниже — экзотическая короткошерстная и плюшевая игрушка. Нарушение работы поискового алгоритма. 
- **P0 — Ошибка загрузки страницы.** В правом верхнем углу виден системный текст: «Ошибка, обновите страницу». Это указывает на сбой фронтенда или потерю связи с бэкендом. 
- **P0 — Нарушение работы фильтра по цене.** В боковом меню применен фильтр по цене «от 1000 до 30000 ₽», однако в результатах выдачи присутствуют объявления, выходящие за рамки этого диапазона (персидский котенок девочка за 40000р и Экзотический короткошерстный котенок)

---

## P1 — Высокий приоритет

- **P1 — Несоответствие фильтра «Возраст» категории.** В категории «Кошки» в фильтре возраста выбран «щенок», хотя дожны ожидать «котенок». Это ошибка локализации или привязки фильтров к категории. 
- **P1 — Смешивание товаров разного типа.** В выдаче котят присутствует «Персидский котенок плюшевый» (игрушка). Это мешает целевому поиску пользователя.
- **P1 — Несоответствие предложенного фильтра основному ГЕО.** Система предлагает быстрый фильтр «Сначала из Можайска», хотя в поиске выбрана «Москва». 

---

## P2 — Средний приоритет

- **P2 — Ошибка геопозиции в выдаче.** В заголовке указано «Котята в Москве», однако самое первое объявление в списке — из Новосибирска. Если не включена опция «по всей России», это баг по гео.
- **P2 — Наложение тултипа на контент.* При наведении курсора на иконку «добавить в избранное» (сердечко) на третьей карточке в первом ряду, тултип «Не показывайте мне похожие объявления» отображается с некорректным позиционированием.
- **P2 — Нарушение навигационной цепочк.** В навигационной цепочке (хлебные крошки) над блоком фильтров отсутствует возможность перехода на верхние уровни категорий. Отображается только конечный путь: Животные -> Кошки, но нет ссылки на «все категории» или «главную», а также неактивны промежуточные звенья. 

---

## P3 — Низкий приоритет

- **P3 — Отсутствие единообразия в отображении даты публикации.** В поисковой выдаче отсутствует единый стандарт отображения времени публикации объявления. Часть карточек содержит информацию о дате (7 дней назад), в то время как остальные только локацию.
